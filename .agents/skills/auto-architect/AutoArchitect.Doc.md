SKILL NAME: AutoArchitect.Doc

DESCRIPTION:
Responsável por gerar documentação técnica completa em Markdown com base na análise previamente realizada pela AutoArchitect.Scan.

---

## PRÉ-CONDIÇÃO

Esta skill assume que:
- O sistema já foi analisado completamente
- Existe um modelo mental consolidado
- Todas as dependências e regras já foram identificadas

---

## MODO DE OPERAÇÃO

Quando esta skill for ativada:

1. NÃO reanalisar superficialmente o sistema
2. Utilizar o conhecimento já consolidado
3. Focar exclusivamente em gerar documentação completa
4. Produzir saída altamente estruturada

---

## FORMATO DE SAÍDA

Gerar UM ÚNICO documento Markdown (.md)

---

## ESTRUTURA OBRIGATÓRIA

### 1. Visão Geral do Sistema
- Descrição
- Objetivo de negócio
- Funcionalidades
- Arquitetura

---

### 2. Arquitetura do Sistema
- Estrutura geral
- Comunicação entre camadas
- Fluxo de dados
- Diagrama textual

---

### 3. Tecnologias Utilizadas
- Linguagens
- Frameworks
- Bibliotecas
- Banco de dados
- Ferramentas externas

---

### 4. Setup do Ambiente
- Pré-requisitos
- Instalação
- Variáveis de ambiente
- Execução local
- Deploy

---

### 5. Frontend
- Estrutura de pastas
- Componentes
- Rotas

- Regras de negócio

---

### 6. Backend
- Estrutura
- Camadas
- Endpoints detalhados
- Validações
- Autenticação
- Regras de negócio

---

### 7. Banco de Dados

Para cada tabela:
- Nome
- Descrição
- Campos
- Relacionamentos

#### Tabelas com cálculo (DETALHAR COMPLETAMENTE):
- Fórmulas
- Execução
- Dependências
- Impacto

---

### 8. Fluxos do Sistema
- Fluxos principais

### 9. Regras de Negócio

### 10. Pontos Críticos

- Cuidados

### 11. Guia de Recriação

Passo a passo:
1. Estrutura inicial

---

## REGRAS

- Ser extremamente detalhado
- Não omitir informações
- Não assumir conhecimento prévio
- Explicar tudo com clareza
- Priorizar completude sobre brevidade

---

## SAÍDA FINAL

- Apenas o Markdown
- Sem comentários fora do documento
- Documento pronto para uso
